# WB Parser

Тестовый проект для вакансии `Python-разработчик` в `VERBA-group`.

[`Тех. задание`](https://docs.google.com/document/d/1eD6om-kZag15XL1mHBqkI50oBYy3Szh0jXZqd3NS37Q/edit?tab=t.0)


## Описание

Ищет товары по поисковому запросу и сохраняет результаты в `results.xlsx` файле,
а так же сохраняет фильтрованные результаты в файл `results_filtered.xlsx`.


## Быстрый старт

1. Склонировать репозиторий
```
git clone https://github.com/adsFreedom/verba_wb_parser
cd verba_wb_parser
```

2. Создать виртуальное окружение
```win10
python -m venv venv
venv\Scripts\activate
```

3. Создать `.env` файл с настройками
```
cp .env_example .env
```
И обязательно заполнить поле `x_wbaas_token`

4. После этого запустить поочерёдно скрипты
```
python 1_load_pages.py
python 2_load_cards.py
python 3_make_xls.py
```

В результате в корне директории будут два файла:
- `results.xlsx` - все результаты
- `results_filtered.xlsx` - результаты отфильтрованные согласно задаче


## Описание работы скриптов

### [`1_load_pages.py`](1_load_pages.py) - получение `pages`

1. При каждом страте создает директорию для сохранения json файлов.
2. Получает общее число товаров
3. Вычисляет кол-во `pages`, которые необходимо загрузить (pages возвращает по 100 товаров)
4. Загружает каждый `pages` и сохраняет как JSON файл 

### [`2_load_cards.py`](2_load_cards.py) - получение информации о каждом товаре `card`

1. Из директории `pages` получает все id товаров, которые нужно загрузить
2. Получает id загруженных товаров из директории `cards`
3. Загружает карточку каждого товара и сохраняет её в JSON

### [`3_make_xls.py`](3_make_xls.py) - Создание `xls` файлов

1. Открывает файлы результатов
2. Для каждого id товара из файлов `pages` находит `card` и создает модель `Goods`
(Если для товара файла `card` - загружается со значениями по умолчанию)
3. Сохраняет `goods` для каждого товара в xls файл
